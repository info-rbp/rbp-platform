# Admin CRUD Scaffold

This document describes the first admin CRUD scaffolding pass.

## What this phase adds

- A generic admin CRUD scaffold page.
- Admin routes for major sidebar destinations.
- Route coverage for nested admin sections.
- A scaffold audit script.

## What this phase does not add

- Backend persistence
- Real CRUD mutations
- Firebase integration
- Authentication beyond the existing local admin guard
- Role-based access control
- File uploads
- Payment workflows
- Legal approval workflows

## Scaffolded admin sections

- Dashboard utility routes
- On-Demand Services
- Managed Services
- Applications
- Operations
- Marketplace
- Membership
- Offers
- Resources
- Help Center
- Site Content
- Settings

## Next implementation step

After this scaffold is merged, the next phase should add reusable admin form primitives:

- Admin page header
- Admin table
- Admin status badge
- Admin empty state
- Admin form shell
- Admin drawer/modal shell
- Admin field renderer

Tiny thrilling stuff. The sort of thing that keeps civilization limping forward.
